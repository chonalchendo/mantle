## Summary
Implemented Story 1 of Issue 30: tag filtering for list-skills command.

## What Was Done
- Added optional `tag: str | None = None` keyword argument to `list_skills()` in `core/skills.py`
- When tag is provided, iterates over skill files, loads each via `load_skill()`, and filters by tag membership; skips unparseable files (NoteParseError/NoteValidationError)
- Modified `list_skills_command()` in `cli/main.py` to accept `--tag` parameter and pass it through to core function
- Added empty-result message: 'No skills matching tag ...' when filtering is active and no skills match
- Added 6 new tests in `tests/core/test_skills.py::TestListSkills` and 3 new tests in `tests/cli/test_skills.py::TestListSkillsCommand`

## Decisions Made
- Optional kwarg on existing function (not a new function), per story spec
- CLI test for tag filtering calls `list_skills_command` from `mantle.cli.main` directly since the command is inline in main.py, not delegated to a cli/skills.py helper
