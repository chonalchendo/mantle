## Summary
Investigated the Claude Code repo for patterns applicable to Mantle, then implemented three high-leverage improvements: tool allowlists, inline context injection, and structured hooks.

## What Was Done
- Added allowed-tools frontmatter to 17 commands (restricting each to minimal tool set)
- Fixed implement.md missing Agent in allowed-tools
- Added inline context injection (!`backtick`) to verify, review, retrospective commands
- Created PostToolUse hook for auto-formatting (ruff, prettier, gofmt, rustfmt)
- Created Stop hook for session-log nudge
- Refactored _register_hooks into generic _ensure_hook helper
- Added 3 new tests for hook registration
- Released v0.7.12

## Decisions Made
- PostToolUse reads file path from stdin JSON (not env vars — confirmed via Claude Code bug #9567)
- Tool allowlists use Bash(mantle *) pattern for CLI commands, with granular patterns only where needed
- Inline context injection only on code-facing commands (verify, review, retrospective) — not design commands

## What's Next
- Test PostToolUse auto-formatting in a real project
- Monitor whether tool allowlists cause any permission issues during workflows
